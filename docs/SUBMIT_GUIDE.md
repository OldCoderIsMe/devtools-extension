## 🚀 下一步操作

### 步骤 1：注册开发者账号（如果还没有）

1. **访问开发者控制台**：
   - 打开：https://chrome.google.com/webstore/devconsole
   - 使用 Google 账号登录

2. **支付注册费**：
   - 一次性支付 $5 USD（约 35 元人民币）
   - 这是 Google 的开发者注册费用，永久有效
   - 支持信用卡、借记卡、PayPal 等支付方式

3. **接受开发者协议**：
   - 阅读并接受 Chrome Web Store 开发者协议

### 步骤 2：上传扩展

1. **进入开发者控制台**：
   - 登录后，点击左上角的"**新建项目**"按钮

2. **上传 ZIP 文件**：
   - 点击"**选择文件**"或拖拽文件
   - 选择：`devtools-suite-v0.1.0.zip`
   - 等待上传完成（可能需要几分钟）

3. **验证上传**：
   - 上传成功后，会显示扩展的基本信息
   - 检查版本号、名称等是否正确

### 步骤 3：填写商店信息

#### 基本信息

**名称**：
```
DevTools Suite
```

**简短描述**（最多 132 字符）：
```
常用开发小工具集合：URL编码、加密哈希、时间戳转换、JSON格式化、文本处理、UUID生成、正则测试、二维码生成、文本对比等。所有计算在本地完成，保护隐私安全。
```

**详细描述**（最多 16,000 字符）：
```
DevTools Suite 是一个功能丰富的 Chrome 浏览器扩展，为开发者提供常用的工具集合。

主要功能：

🔗 URL 编码/解码
- URL 编码和解码
- 支持批量处理

🔐 加密/哈希工具
- Hash 算法：MD5、SHA1、SHA256、SHA512
- Base64 编码/解码
- AES 加密：AES128/AES256 加密/解密
- HMAC：HMAC-MD5、HMAC-SHA1、HMAC-SHA256、HMAC-SHA512
- 支持大小写切换

⏰ 时间戳转换
- 时间戳（秒/毫秒）→ 本地时间
- 日期时间 → 时间戳（毫秒）

📄 JSON 格式化工具
- JSON 格式化（美化）
- JSON 压缩
- JSON 验证
- JSON 转义/反转义

📝 文本处理工具
- 文本去重（按行）
- 文本排序（按行，支持升序/降序）
- 大小写转换（大写/小写/首字母大写/驼峰命名）
- 文本统计（字符数、单词数、行数、段落数）

🆔 UUID/随机字符串生成器
- UUID v4 生成
- 随机字符串生成（可配置字符集和长度）
- 随机密码生成（可配置字符类型）
- 支持批量生成

🔍 正则表达式测试工具
- 正则匹配测试
- 支持全局、忽略大小写、多行等标志
- 替换预览功能
- 显示匹配结果和分组

📱 二维码生成器
- 文本/URL 转二维码
- 可配置尺寸（200-500px）
- 支持下载二维码图片

🔄 文本差异对比工具
- 按行对比
- 按字符对比
- 高亮显示新增/删除/未改变的内容
- 显示统计信息

隐私安全：
- 所有计算都在本地浏览器中完成
- 不会上传任何数据到服务器
- 保护您的隐私安全

技术栈：
- Vue 3 + TypeScript
- 所有加密算法使用 crypto-js 库
- 二维码生成使用 qrcode 库
```

### 步骤 4：设置定价和分发

1. **定价**：
   - 选择"**免费**"

2. **分发范围**：
   - 选择"**公开**"（所有人都可以安装）
   - 或者选择特定地区（如果需要）

3. **可见性**：
   - 选择"**公开可见**"

### 步骤 5：提交审核

1. **检查所有信息**：
   - ✅ 名称和描述正确
   - ✅ 图标已上传
   - ✅ 截图已上传（至少 1 张）
   - ✅ 分类和语言已选择
   - ✅ 隐私政策已说明

2. **提交审核**：
   - 点击"**提交审核**"按钮
   - 确认提交

3. **等待审核**：
   - 审核时间：通常 **1-3 个工作日**
   - 高峰期可能需要更长时间
   - 审核状态会在开发者控制台显示

### 步骤 6：审核通过后

1. **扩展自动发布**：
   - 审核通过后，扩展会自动发布到 Chrome Web Store
   - 用户可以在商店搜索并安装

2. **分享链接**：
   - 在开发者控制台可以获取扩展的商店链接
   - 可以分享给其他人

## ⚠️ 常见审核问题

### 如果审核被拒绝

1. **查看拒绝原因**：
   - 在开发者控制台查看详细的拒绝原因
   - Google 会说明具体问题

2. **常见拒绝原因**：
   - 功能不完整或无法正常工作
   - 权限申请过多或不合理
   - 商店描述不准确
   - 截图不符合要求
   - 隐私政策缺失

3. **修改后重新提交**：
   - 根据反馈修改问题
   - 重新上传 ZIP 包（如果需要）
   - 重新提交审核
